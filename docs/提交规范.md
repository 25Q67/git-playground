# Git 提交规范

## 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 1. 类型（Type）

| 类型       | 说明                       |
| ---------- | -------------------------- |
| `feat`     | 新功能                     |
| `fix`      | Bug 修复                   |
| `docs`     | 文档更新                   |
| `style`    | 代码格式调整（不影响功能） |
| `refactor` | 重构代码                   |
| `perf`     | 性能优化                   |
| `test`     | 添加/修改测试              |
| `chore`    | 构建工具或辅助工具的变动   |

### 2. 范围（Scope）可选

描述提交影响的范围，如：`docs`, `profiles`, `api` 等。

### 3. 主题（Subject）

- 使用简洁的描述
- 动词开头，小写字母开头
- 不超过 50 个字符

### 4. 正文（Body）可选

详细说明本次提交的动机和变更内容，每行不超过 72 个字符。

### 5. 页脚（Footer）可选

- 破坏性变更标注 `BREAKING CHANGE`
- 关联 Issue 使用 `Closes #123`

## 示例

```
feat(profiles): 添加用户头像上传功能

实现了用户可以在个人简介页面上传头像的功能，
支持 jpg、png 格式，最大 2MB。

Closes #45
```

```
fix(docs): 修正 Git 教程中的拼写错误

将 "rebase" 正确拼写为 "rebase"
```

## 注意事项

1. 每次提交应该是原子性的（只包含一个逻辑变更）
2. 提交信息使用中文或英文，保持一致性
3. 不要在提交信息中包含无用信息如 "fixed typo"
4. 提交前检查代码，确保没有调试输出
